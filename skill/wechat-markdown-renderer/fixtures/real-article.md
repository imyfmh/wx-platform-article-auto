# 把周报写成真正有人愿意读的文章

每周写周报最常见的问题，不是缺内容，而是内容没有结构。读者看到一长串事项罗列，很难快速抓住重点，也很难判断哪些信息值得继续跟进。

## 先给结论，再给过程

如果一篇周报只能让读者记住一句话，那这句话应该出现在最前面。

> 好的周报不是“我做了什么”，而是“这周发生了什么变化，以及它意味着什么”。

### 一个简单模板

1. 本周最重要的结果
2. 对业务产生的影响
3. 当前风险与阻塞
4. 下周明确动作

## 例子

| 项目 | 变化 | 影响 |
| --- | --- | --- |
| 注册流程 | 表单缩短到 3 步 | 完成率上升 |
| 帮助中心 | 新增 FAQ | 客服压力下降 |

### 代码片段

```ts
export function summarize(items: string[]) {
  return items.filter(Boolean).join(' / ')
}
```
